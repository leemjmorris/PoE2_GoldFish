# PoE2 GoldFish — 프로젝트 로드맵

## 목표
PoE Overlay2, Sidekick과 같은 Path of Exile 2 전용 인게임 오버레이 프로그램 개발.
**기술 스택: C# / WPF / .NET 8 / Windows**

---

## 기능 목록 및 구현 상태

| # | 기능 | 상태 | 핵심 파일 |
|---|------|------|-----------|
| 1 | 메모장 오버레이 | ✅ 완료 | `Features/Memo/` |
| 2 | 스크린샷 오버레이 | ✅ 완료 | `Features/Screenshot/` |
| 3 | 아이템 시세 조회 (Trade) | ✅ 완료 | `Features/Trade/` |
| 4 | 패시브 스킬 트리 오버레이 | ⬜ 미구현 | — |

---

## 기능 상세 설명

### 1. 메모장 오버레이 ✅
- 인게임 중 메모 작성 및 저장
- 마지막 종료 시점의 메모를 다음 실행 때도 복원 (영속성)
- 핫키로 오버레이 토글

### 2. 스크린샷 오버레이 ✅
- 지정 경로의 스크린샷을 썸네일로 오버레이에 표시
- 마우스 호버 시 이미지 확대 미리보기
- 현재 화면 해상도에 대응하여 이미지 크기 조정

### 3. 아이템 시세 조회 (Trade) ✅
- 인게임에서 아이템 복사 → 클립보드 파싱 → 자동 시세 조회
- 아이템 옵션(stat) 개별 체크/해제
- 최솟값 / 최댓값 / 오차 범위 세부 설정
- PoE2 공식 Trade API 연동

### 4. 패시브 스킬 트리 오버레이 ⬜
- PoB2 (Path of Building 2) 공유 URL 입력
- URL에 인코딩된 패시브 트리 노드 디코딩
- 노드 목록 또는 트리 시각화를 오버레이로 표시
- **구현 난이도 가장 높음** — PoB2 데이터 포맷 분석 필요

---

## 개발 순서 및 이유

```
1단계: 메모장  →  오버레이 인프라 구축 (투명 윈도우, TopMost, 핫키, 드래그)
2단계: 스크린샷  →  1단계 프레임워크 재활용, 디렉토리 감시 + 이미지 UI 추가
3단계: 아이템 시세  →  클립보드 파싱 + Trade API 연동 (작업량 많음, 핵심 기능)
4단계: 패시브 트리  →  PoB2 데이터 디코딩 + 트리 시각화 (가장 복잡, 마지막)
```

## 공통 오버레이 인프라 (Core)

모든 기능이 공유하는 기반 컴포넌트:
- `Core/HotkeyManager.cs` — 전역 핫키 등록/해제
- `Core/ClipboardListener.cs` — 클립보드 변경 감지
- `Core/AppSettings.cs` — 설정 JSON 저장/로드
- `MainWindow.xaml.cs` — 시스템 트레이 아이콘, 기능 간 조율
